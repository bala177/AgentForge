🧠
ReAct Pattern
Reason → Act → Observe → Repeat. The LLM thinks step-by-step, calls tools, observes results, then synthesizes a final answer.
📜
System Prompt
Lists all 13 tools with descriptions. Forces JSON responses only. Includes rules: must use tools for real-world data, never hallucinate.
🔨
JSON Actions
LLM responds with structured JSON: {action: "tool_call", tool, input} or {action: "answer", text}
🔌
Keyword Fallback
If Ollama is offline or LLM errors out, seamlessly falls back to regex-based keyword matching. Always works, no AI needed.
🔁
Follow-up Synthesis
After tool execution, the raw result is sent back to the LLM with a follow-up prompt. LLM synthesizes a human-friendly answer.
🔎
Auto-detect Model
On startup, queries Ollama for installed models and picks the best one (llama3 > mistral > tinyllama). No manual config needed.
🛠
Action Normalization
Handles LLM quirks: if the model returns the tool name as the action instead of "tool_call", it’s auto-corrected. Empty answers trigger a retry nudge.
📈
Memory & Steps
Every tool call is logged to memory. Responses include full step traces (thought, tool, input, result) visible in the expandable steps panel.
